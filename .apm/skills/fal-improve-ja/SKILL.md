# 日本語の推敲と監査

このスキルは、書き上がった日本語の文章への推敲・監査のパスです。目標とする状態は fal-write-ja が定めます。監査の前に fal-write-ja の本文を読み、[用例集](./references/examples.md) を通読してパターンを把握してください。対象は、直近の作業で書いた日本語、またはユーザーが指定した文章です。

## 監査者の姿勢

用法の失敗（fal-write-ja §1〜§3）は、生成の土台となるパターンそのものが失敗の源であるため、書いた本人には自然に見えます。可能なら、推敲・監査は書いたセッションとは別の文脈で行います。新しいセッションやサブエージェントに、このスキルと fal-write-ja、対象の文章だけを読ませます。書いたときの文脈を共有しない読み手のほうが、確実に見つけられます。

修正の加減は fal-write-ja §7 に従います。定着した術語、プロジェクトで定義済みの語、ユーザー自身が使っている語まで置き換えるのは過剰修正です。

## 手順

少なくないケースが文法の検査は通ってしまうので、語の単位を中心に見ます。§ の参照は fal-write-ja 本文の節を指します。

1. 語の照合: 用例集の「誤用されやすい語の一覧」にある語を、対象の文章から機械的に探します。見つけた語は機械的に置き換えるのではなく、項目の説明で場面を確かめてから言い直します。一覧は頻度の高い失敗を先に拾うための補助であり、照合を終えても、次の用例の確認は省略できません。
2. 用例の確認: 文意を支える語（漢語・専門用語・比喩）ごとに、その語がこの意味で、この種類の文章の中で使われているのを読んだ覚えがあるかを問います（§1）。複合語は、まず定着した型（§2）に当たるかと、省いた語を補うと戻し方が一通りに決まるかを問い、型に当たらない連結だけ、連結そのものの用例を問います。述語は主語・目的語との組み合わせまで問います。読んだ覚えのない語は、造語・別の場面からの流用・英語の語義の直訳のいずれかを疑って言い直します（§1〜§3）。プロジェクトや分野で定着した術語は、この確認の対象外です（§7）。
3. 逆翻訳: 引っかかる表現を英語に訳し戻し、自然な英語の慣用句が現れたら直訳を疑って言い直します（§3）。
4. 平易化: 日常語だけで言い直しても失うものがないなら、平易な側を採ります。言い直せないなら、その専門用語は仕事をしています。
5. 場面想定: 雑談でしか言わない言い回しは口語の混入です。
6. 構文と修辞の点検: 行為を分岐させない条件形、体言止め、近くで重なる比喩、「X ではなく Y」の多用を探して書き直します（§4〜§5）。
7. 記号の点検: コロン・ダッシュ・強調の鉤括弧を機械的に探し、英語の用法のものを書き直します（§6）。

## 編集の規律

- 監査を通った文は、字句も改行もそのまま残します。目的のない書き換えは差分のノイズになり、書き直した文には新たな失敗が入り込みます。
- 言い直しは fal-write-ja の規則に全面的に従います。特に、置き換え先の語にも用例の確認（§1）を適用します。
